---
title: トランザクション
---

トランザクションとは DB における 1 つの作業単位であり, begin から始まり, commit / rollback で完結する.
トランザクションは ACID 特性を満たす必要がある.

| 特性                 | 内容                                                                                      |
| -------------------- | ----------------------------------------------------------------------------------------- |
| 原子性 - Atomicity   | トランザクションは全ての処理を完了するか, 何も行わないかのいずれかであること.             |
| 一貫性 - Consistency | トランザクションが完了した時点で, データベースは一貫した状態であること.                   |
| 隔離性 - Isolation   | 同時に実行されるトランザクションは互いに干渉しないこと.                                   |
| 耐久性 - Durability  | トランザクションが完了した後は, たとえシステム障害が発生しても, その結果が失われないこと. |

## ロック

専有ロック: トランザクションがデータを専有し, 他のトランザクションはそのデータにアクセスできない.
共有ロック: トランザクションはデータを読み取ることができるが, 他のトランザクションも同様に読み取ることができる. 書き込みはできない.

### 2 相ロック方式

あるトランザクションでデータに対する操作時に, 排他対象の資源に対して一斉にロックを取得し, その後一斉にロックを解放する方式.


## 隔離性水準

SQL-92 にて定義された機能拡張.
ダーティリード, ノンリピータブルリード, ファントムリードが発生するか否かが変化する.

| 隔離性水準       | ダーティリード | ノンリピータブルリード | ファントムリード |
| ---------------- | -------------- | ---------------------- | ---------------- |
| READ UNCOMMITTED | 発生する       | 発生する               | 発生する         |
| READ COMMITTED   | 発生しない     | 発生する               | 発生する         |
| REPEATABLE READ  | 発生しない     | 発生しない             | 発生する         |
| SERIALIZABLE     | 発生しない     | 発生しない             | 発生しない       |

### ダーティリード

トランザクションが他のトランザクションの未コミットの内容を読み取ること.

### ノンリピータブルリード

トランザクション中に同じクエリを再実行した際に、他のトランザクションによって変更されたデータが取得されること.

### ファントムリード

トランザクション中に同じクエリを再実行した際に、他のトランザクションによって挿入されたデータが取得されること.
