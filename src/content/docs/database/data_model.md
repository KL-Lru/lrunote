---
title: データモデル
---

データ構造を矛盾なく管理できるように整理するための概念としてデータモデルという考え方がある.
考え方であるため, 様々なものが提唱されている.

基本的には「抽象的な現実世界」から「物理的なデータ構造や格納形態」へと落とし込む形が取られる.

## JIPDEC データモデル

| モデル           | 内容                                                                           |
| ---------------- | ------------------------------------------------------------------------------ |
| 概念データモデル | 利用者の取り扱うデータや関係を示す抽象的なモデル                               |
| 論理データモデル | 概念データモデルを基に、データの構造や関係を機械処理可能な形態に表現したモデル |
| 物理データモデル | 論理データモデルを基に、データの物理的な格納構造などを考慮して作成されるモデル |

## ANCI/X3/SPARC 3 層スキーマアーキテクチャ

| 段階         | 内容                                                                                      |
| :----------- | :---------------------------------------------------------------------------------------- |
| 外部スキーマ | 利用者から直接見えるスキーマ. View 定義やアクセス可能なデータへのフィルタリングなどを行う |
| 概念スキーマ | どのようなデータが格納されるかを指す抽象的なデータの構造と関係を示すスキーマ.             |
| 内部スキーマ | 物理的な表現や格納方法を定義するスキーマ. データ型やインデックス定義などを含む            |
